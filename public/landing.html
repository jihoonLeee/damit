Why Damit

현장 기록부터 시작합니다

사진과 이유를 먼저 남기고, 그다음 작업 건 연결과 견적 정리로 이어지는 구조입니다.

설명 문장을 바로 만듭니다

고객에게 보낼 설명 초안과 확인 링크를 준비해 현장 이후 후속 연락을 빠르게 처리합니다.

합의 기록을 나중에도 다시 봅니다

금액, 채널, 메모, 타임라인을 남겨서 분쟁 대응과 내부 공유가 쉬워집니다.

Workflow

작업 흐름은 다섯 단계로 이어집니다

1. 현장 문제 기록

사진과 사유를 빠르게 남겨서 현장 상황을 놓치지 않습니다.

2. 작업 건 연결

새 작업 건을 만들거나 기존 작업 건에 연결해 하나의 케이스로 관리합니다.

3. 변경 견적 정리

기본 금액과 변경 금액을 비교하면서 설명 근거를 만듭니다.

4. 설명과 확인 정리

설명 초안, 고객 확인 링크, 합의 기록, 타임라인을 한 화면에서 이어서 관리합니다.
