# Brainstorm Vision

## 역할
- 새로운 아이디어를 넓게 발산한다.
- 아직 feasibility, market, risk 기준으로 조기 탈락시키지 않는다.
- 기능보다 문제 장면, 사용자, 기대 가치에 집중한다.

## 역할 경계
### 이 Skill 이 하는 일
- 서로 다른 방향의 후보를 최소 5개 이상 만든다.
- 각 후보가 해결하는 핵심 문제를 선명하게 적는다.
- 왜 이 아이디어가 지금 의미 있는지 가설을 만든다.

### 이 Skill 이 하지 않는 일
- 구현 가능성 최종 판단
- 시장성 최종 판단
- 후보 탈락 결정
- 최종 우승안 선정

## 언제 사용하나
- 아이디어가 아직 없거나 너무 좁을 때
- 기존 아이디어를 다시 발산해야 할 때
- PRD 이전 단계에서 선택지를 넓혀야 할 때

## 선행 조건
- 사용자 목표 또는 문제 맥락이 최소한 한 문장으로 정리돼 있어야 한다.
- 시간/예산/운영 제약이 있으면 함께 받아야 한다.

## 입력
### 필수 입력
- 사용자 목표와 배경
- 시간, 예산, 기술, 운영 제약

### 선택 입력
- 타깃 업종 또는 사용자 힌트
- 기존 아이디어 문서: docs/product/IDEA_POOL.md
- 이전 선정 결과: docs/product/IDEA_FINAL.md

### 입력이 부족할 때 기본 가정
- 사용자는 빠른 검증과 현실적 수익화를 원한다.
- 1인 또는 소규모 팀 기준으로 시작한다.

## 출력
- docs/product/IDEA_POOL.md

## 출력 계약
각 아이디어는 최소 아래 항목을 포함한다.
- 아이디어명
- 한 줄 설명
- 핵심 문제
- 핵심 사용자
- 기대 가치
- 왜 지금 의미 있는지
- 다음 단계가 검증할 전제

## 판단 원칙
- 새로운 기능보다 새로운 문제 프레이밍을 우선한다.
- 비슷한 후보라도 타깃이나 가치가 다르면 남긴다.
- 사용자의 수익, 시간, 불안 감소 중 하나와 연결되는 후보를 우선한다.
- 아직 `좋아 보이는가`보다 `문제가 선명한가`를 본다.

## 작업 절차
1. 사용자 목표와 제약을 3~5줄로 요약한다.
2. 후보를 문제 장면별로 최소 5개 이상 만든다.
3. 각 후보를 `문제/사용자/가치` 구조로 정리한다.
4. 중복 후보는 합치되 관점 차이는 보존한다.
5. practical 이 검증할 전제를 후보별로 2~3개 남긴다.
6. IDEA_POOL 에 vision 섹션으로 기록한다.

## 품질 기준
- 후보가 서로 충분히 다르다.
- 기능 목록이 아니라 문제와 사용자가 먼저 적혀 있다.
- 경쟁이나 난이도로 조기 탈락시키지 않았다.
- 다음 단계가 바로 검토할 전제가 남아 있다.

## Self-Review
- 너무 비슷한 후보만 반복하지 않았는가
- 사용자보다 솔루션이 먼저 나오지 않았는가
- 가치가 추상적이지 않은가
- 다음 단계 질문거리가 남아 있는가

## 금지
- 구현 난이도만 보고 아이디어를 제거하지 않는다.
- 경쟁 존재만으로 탈락시키지 않는다.
- final 선택처럼 한 후보만 밀어붙이지 않는다.
- 다른 brainstorm skill 의 역할까지 대신하지 않는다.

## 다음 단계 handoff
- 다음 Skill: `brainstorm-practical`
- handoff packet:
  - 검증할 전제 2~3개
  - 타깃 사용자
  - 반복되는 문제 장면
  - 비슷하지만 남겨둔 대안 후보

## 예시 입력
```md
- 목표: 본업과 병행할 수 있는 월 반복매출형 사이드 프로젝트를 찾고 싶다.
- 제약: 혼자 개발, 주말 중심, 6주 안에 MVP 필요.
- 타깃 힌트: 로컬 홈서비스 사장님 또는 소상공인.
- 기존 메모: 빠른 유료 검증이 중요하고 거대 플랫폼과 정면충돌은 피하고 싶다.
```

## 예시 출력
```md
### 아이디어 A. 현장 음성비서
- 한 줄 설명: 당근/전화/카톡 문의를 음성 메모로 바로 구조화해 견적 가능한 상태로 바꾸는 도구
- 핵심 문제: 현장형 사장님은 통화 후 기록을 안 남겨 견적이 늦어진다.
- 핵심 사용자: 입주청소, 에어컨 청소 업자
- 기대 가치: 메모 스트레스 감소, 견적 응답 속도 향상
- 왜 지금 의미 있는지: 인입 채널은 이미 많지만 후처리 레이어는 비어 있다.
- 다음 단계가 검증할 전제: 업종별 반복 질문을 템플릿화할 수 있는가
```
