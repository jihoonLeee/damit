# Skill Review Checklist

## 기본
- [ ] name 이 역할을 잘 설명한다
- [ ] description 이 호출 시점을 설명한다
- [ ] 역할 범위가 명확하다
- [ ] 비범위가 있다

## 입출력
- [ ] 입력이 정의되어 있다
- [ ] 출력 파일 경로가 있다
- [ ] 산출물 형식이 정해져 있다

## 연결성
- [ ] 선행 조건이 있다
- [ ] 핸드오프 대상이 있다
- [ ] 다음 단계가 명확하다

## 품질
- [ ] 금지사항이 있다
- [ ] 중복 Skill과 충돌이 없다
- [ ] 역할이 너무 넓지 않다
- [ ] 검토 기준이 있다

## 개선 필요 여부
- [ ] description 재작성 필요
- [ ] 역할 분리 필요
- [ ] 병합 필요
- [ ] 출력 형식 보강 필요
- [ ] 핸드오프 보강 필요
