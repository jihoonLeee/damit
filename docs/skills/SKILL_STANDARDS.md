# SKILL STANDARDS

## 목적
이 문서는 저장소의 모든 `skills/*/SKILL.md`가 같은 품질 기준을 따르도록 하는 운영 표준이다.
각 Skill 은 읽는 즉시 아래 다섯 가지가 분명해야 한다.
- 무엇을 한다
- 무엇을 하지 않는다
- 무엇을 입력으로 받는다
- 무엇을 출력으로 남긴다
- 다음 단계에 무엇을 넘긴다

## 기본 원칙
- Skill 은 하나의 주 책임만 가져야 한다.
- Skill 은 입력, 출력, 금지사항, handoff 가 반드시 있어야 한다.
- Skill 은 문서를 만들거나 갱신할 때 파일 경로까지 명시해야 한다.
- Skill 은 추상적인 조언이 아니라 반복 가능한 작업 절차를 제공해야 한다.
- Skill 은 다음 단계가 바로 이어받을 수 있는 handoff packet 을 남겨야 한다.

## 필수 구성요소
모든 SKILL.md 는 최소 아래 섹션을 포함한다.

1. Frontmatter
- `name`
- `description`

2. 역할
- 이 Skill 의 주 책임
- 이 Skill 이 다루는 범위
- 이 Skill 이 다루지 않는 범위

3. 언제 사용하나
- 트리거 조건
- 선행 문서 또는 선행 단계

4. 입력
- 필수 입력 문서
- 선택 입력 문서
- 입력이 없을 때의 기본 가정

5. 출력
- 산출물 파일 경로
- 산출물마다 반드시 포함할 항목
- 문서가 아닌 경우 결과 요약 형태

6. 작업 절차
- 3~7단계 정도의 반복 가능한 순서
- 필요 시 의사결정 포인트 포함

7. 판단 원칙
- 우선순위를 어떻게 정하는지
- trade-off 를 어떻게 결정하는지
- 경계가 겹칠 때 어떤 기준으로 나누는지

8. 품질 기준
- 완료로 보기 위한 조건
- self-check 항목
- 흔한 실패 패턴 방지 문구

9. 금지
- 범위 확장 금지
- 무근거 추정 금지
- 다른 Skill 역할 침범 금지

10. Handoff
- 다음 Skill 또는 다음 단계
- 반드시 넘겨야 하는 정보 묶음

## 이름 규칙
- 이름은 역할을 드러내야 한다.
- 저장소 디렉터리명과 frontmatter `name`은 가능하면 일치시킨다.
- 모호한 이름을 피한다.

좋은 예시
- `feature-spec`
- `qa-review`
- `pm-brainstorm`

나쁜 예시
- `helper`
- `worker-1`
- `agent`

## Description 규칙
설명은 아래 세 요소를 모두 포함한다.
- 언제 호출되는지
- 무엇을 생산하는지
- 어디에 반영하는지

좋은 예시
- `PRD와 UX 설계를 구현 가능한 기능/데이터/API 명세로 변환해 docs/specs 아래 문서를 작성하는 스킬`

## 입력 규칙
- 필수 입력과 선택 입력을 구분한다.
- 문서 경로는 저장소 기준으로 적는다.
- 입력이 없을 때 어떤 가정을 둘지 적는다.

## 출력 규칙
- 출력 파일 경로를 구체적으로 적는다.
- 출력 형식을 고정할 수 있으면 최소 구조를 적는다.
- 기존 문서를 갱신하는지 새 문서를 만드는지 명시한다.

## 판단 원칙 규칙
아래 질문에 답할 수 있어야 한다.
- 이 Skill 은 어떤 기준으로 결정을 내리는가
- 어떤 경우에 더 좁히고, 어떤 경우에 더 넓히는가
- 무엇을 우선하고 무엇을 포기하는가

## 품질 기준 규칙
아래 질문에 답할 수 있어야 한다.
- 언제 `완료`라고 말할 수 있는가
- 어떤 누락이 있으면 미완료인가
- 다음 단계가 질문 없이 이어받을 수 있는가

## 금지 규칙
모든 Skill 은 최소 아래 금지사항을 검토한다.
- 합의되지 않은 범위 확장
- 역할 경계 침범
- 입력 없이 결과를 단정
- handoff 없이 종료

## Handoff Packet 규칙
모든 Skill 은 다음 단계에 아래 중 필요한 것을 남긴다.
- 핵심 결정 사항
- 미결정 사항
- 리스크
- 제외한 항목
- 다음 단계가 먼저 확인해야 할 문서

## 2차 검증 기준
2차 검증을 통과한 Skill 은 아래를 만족해야 한다.
- 입력/출력/금지/handoff 가 모두 있다.
- 의사결정 원칙이 적혀 있다.
- self-check 또는 품질 기준이 있다.
- 다음 단계가 바로 실행 가능한 handoff packet 이 있다.
- 역할이 다른 Skill 과 명확히 구분된다.
