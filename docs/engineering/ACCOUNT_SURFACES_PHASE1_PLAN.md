# ACCOUNT SURFACES PHASE 1 PLAN

## 목적

운영형 제품 표면을 아래처럼 명확히 나눈다.

- `/start` : 사장님 시작하기 / 회원가입 입구
- `/login` : 이메일 로그인
- `/home` : 운영 허브
- `/account` : 사장님 마이페이지
- `/ops` : owner 운영 콘솔
- `/admin` : 시스템 관리자 전용 내부 표면

## 이번 배치 범위

### 1. 시작하기 표면 추가

- 랜딩에서 바로 `/login`으로 보내는 흐름 외에, 명시적인 `/start`를 만든다.
- 첫 로그인 시 회사 생성이 곧 회원가입이라는 점을 사용자가 이해할 수 있어야 한다.

### 2. 사장님 마이페이지 추가

- 프로필
- 현재 회사
- 멤버/초대 상태
- 보안/세션 운영 상태
- 운영 화면 이동 동선

### 3. 운영 홈 역할 정리

- `/home`은 “무엇을 먼저 해야 하는지”에 집중
- `/account`는 “내 계정 / 내 회사 / 내 팀”에 집중

### 4. 시스템 관리자 표면 분리

- `/admin`은 owner 화면과 분리된 내부 전용 표면
- 다중 회사, 사용자, 로그인 흐름, 감사 로그, 전역 데이터 탐색에 집중
- owner 권한만으로는 접근 불가

## 구현 원칙

- 기존 로그인/초대/회사 전환 흐름은 유지한다.
- 새 화면은 기존 세션 인증과 company context를 재사용한다.
- 시스템 관리자 권한은 별도 allowlist로 분리한다.
- `/ops`는 owner 회사 운영 콘솔, `/admin`은 시스템 운영 콘솔로 분리한다.

## 검증 기준

- 정적 라우트가 모두 정상 제공된다.
- 로그인 후 `/home`, `/account`, `/ops`, `/admin` 권한 분기가 의도대로 동작한다.
- owner와 system admin의 표면이 혼동되지 않는다.
