# APP_STAGE_POLISH_UX

## UX principle

- 한 단계 화면은 "지금 할 일"이 hero 아래 두 번째 시선 안에 들어와야 한다.
- CTA는 현재 카드 안쪽에 숨어 있지 않고, 단계 전체를 대표하는 안내로 보여야 한다.
- 완료 기준은 사용자를 재촉하는 문장이 아니라 "여기까지 하면 충분하다"로 읽혀야 한다.

## Stage focus card

각 stage 화면에서 `case-focus-card` 아래에 새 `stage-action-card`를 둔다.

### Quote

- 상태 예시: `금액 필요`, `정리됨`
- 제목:
  - 금액 미저장: `변경 금액을 먼저 저장해 주세요`
  - 금액 저장됨: `견적 정리는 끝났습니다`
- 본문:
  - 금액과 추가 범위를 먼저 분명히 하면 이후 설명과 확인이 흔들리지 않는다는 안내
- 완료 기준:
  - `변경 견적이 저장되고 범위 문장이 자연스럽게 읽히면 충분합니다.`
- CTA:
  - 1차: `변경 견적 카드 보기`
  - 2차:
    - 금액 없음: `현장 기록 다시 보기`
    - 금액 있음: `설명 초안 단계로 이동`

### Draft

- 상태 예시: `초안 필요`, `초안 준비됨`
- 제목:
  - 초안 없음: `고객에게 보낼 문장을 먼저 만들어 주세요`
  - 초안 있음: `설명 초안이 준비됐습니다`
- 본문:
  - 고객에게 바로 보낼 문장을 빠르게 만드는 단계라는 점을 강조
- 완료 기준:
  - `복사해서 바로 보낼 수 있는 문장이 있으면 충분합니다.`
- CTA:
  - 1차: `설명 초안 카드 보기`
  - 2차:
    - 초안 없음: `변경 견적 단계로 돌아가기`
    - 초안 있음: `확인과 합의 단계로 이동`

### Confirm

- 상태 예시: `마무리`, `보류`, `완료`
- 제목:
  - 확인 링크 없음: `고객 확인 링크 또는 합의 기록을 정리해 주세요`
  - 링크 있음: `고객 반응과 합의 상태를 함께 정리해 주세요`
  - 보류: `답변을 기다리는 상태입니다`
  - 합의 완료: `합의가 완료된 작업 건입니다`
- 완료 기준:
  - 링크 발급/열람/합의 상태 중 마지막 상태를 한 줄로 설명할 수 있으면 충분
- CTA:
  - 1차: 현재 집중 카드 보기
  - 2차:
    - 링크 없음: `합의 기록 카드 보기`
    - 링크 있음: `타임라인 확인`
    - 완료: `작업 요약 다시 보기`

## Mobile polish

- stage 화면에서는 상단 focus card 두 장만 먼저 강하게 보이게 한다.
- `case-progress-card`와 `detail-summary`는 모바일에서 아래로 밀리고, CTA 카드가 먼저 보이게 한다.
- CTA 버튼은 세로 stack으로 고정한다.
