## 목적

`/app`의 모바일 시각 검수 캡처가 실제 사용 화면을 제대로 보여주지 못하는 문제를 해결한다.

## 현재 문제

- `review=detail` 캡처에 상단 운영 요약 영역이 그대로 남아 있어, 실제 상세 화면보다 대시보드 상단만 크게 보인다.
- `review=agreement` 캡처는 핵심 카드보다 프레임 밖 또는 빈 화면이 잡히는 경우가 있다.
- 모바일 review 모드의 카드 간격과 패딩 리듬이 메인 제품 흐름보다 덜 정리돼 보인다.

## 이번 배치 목표

1. review 모드에서 불필요한 상단 영역을 제거한다.
2. `detail`, `agreement`, `copy` review 모드가 각자 한 화면의 목적을 분명히 드러내게 만든다.
3. `agreement` review 캡처가 빈 화면 없이 안정적으로 재현되게 만든다.
4. 모바일 상세 카드 간격을 메인 UI 리듬과 맞춘다.

## 구현 원칙

- review 모드는 실제 제품 UX를 해치지 않고, 검수용 프레이밍만 별도 최적화한다.
- `agreement` review는 합의 카드 자체를 명확히 보여주는 데 집중한다.
- `detail` review는 상세 흐름 상단 핵심 카드만 보여주고 운영 대시보드 성격의 상단 영역은 숨긴다.
- 수정 후에는 시각 검수 스크립트와 핵심 회귀 테스트를 다시 돌린다.
