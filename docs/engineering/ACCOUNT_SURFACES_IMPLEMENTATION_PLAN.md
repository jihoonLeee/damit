# ACCOUNT SURFACES IMPLEMENTATION PLAN

## 목표

로그인 이후 제품 흐름을 아래처럼 명확히 나눈다.

- `시작하기 / 가입`
- `로그인`
- `운영 홈`
- `사장님 마이페이지`
- `업무 워크스페이스`
- `owner 운영 콘솔`
- `시스템 관리자 전용 표면(후속)`

## Phase 1

### 1. 시작하기 진입면 추가

- `/start`
- 첫 사용자를 위한 설명형 온보딩
- 실제 인증은 기존 매직 링크 재사용

### 2. 로그인/가입 카피 재정리

- `/login`에서 `로그인`과 `처음 시작`의 차이를 분명하게 표시
- 첫 로그인 후 회사 생성은 회원가입 완료 단계로 재정의

### 3. 운영 홈 역할 정리

- `/home`은 “계정 허브”가 아니라 “운영 허브”
- `/account`와 역할 충돌이 없도록 재정리

### 4. 사장님 마이페이지 추가

- `/account`
- 내 프로필
- 회사 기본 정보
- 멤버/초대
- 세션/보안 상태
- 향후 요금/플랜 자리

## Phase 2

### 5. owner 운영 콘솔과 시스템 관리자 표면 분리

- `/ops`는 owner 회사 범위 운영 콘솔로 한정
- `/admin`은 시스템 관리자 전용으로 설계 시작

## PM 게이트

### G1

- 사용자가 `회원가입 / 로그인 / 운영 홈 / 업무 화면` 차이를 헷갈리지 않는가

### G2

- owner가 자기 회사 설정과 팀 관리를 `/account`에서 자연스럽게 이해하는가

### G3

- `/ops`와 `/admin`의 권한/정보 범위가 명확히 분리되는가
