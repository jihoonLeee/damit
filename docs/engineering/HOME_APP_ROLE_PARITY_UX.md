# HOME_APP_ROLE_PARITY_UX

## UX principle

- `/home`은 판단과 이동.
- `/app`은 실행과 기록.

## /home direction

- 상단 카피에서 "운영 홈"의 책임을 먼저 말한다.
- 화면 중간에 `여기서 하는 일 / 작업 허브에서 하는 일`을 대비시킨다.
- 주요 CTA는 `/app/capture`이되, "업무 처리"보다 `작업 허브 열기`처럼 화면 역할이 느껴지는 문구를 쓴다.

## /app direction

- overview 화면은 `selected case execution hub`처럼 읽혀야 한다.
- hero/note/overview hub 카피에서 "회사 전환, 초대, 세션, 운영 점검은 홈/ops"라고 분리한다.
- overview 안에 `/home` 복귀 링크를 넣어 역할 차이를 다시 강화한다.

## Mobile rule

- `/home`에서는 운영 출발점 설명이 fold 위에서 보인다.
- `/app`에서는 stage route card보다 역할 설명과 home return action이 먼저 보인다.
