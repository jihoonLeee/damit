# PRODUCTION_VERSION_PLAN

## 문서 목적

- 현재 파일럿용 MVP를 넘어 실제 유료 배포 가능한 `v1` 제품 기준을 정의한다.
- `무엇을 더 만들지`보다 `무엇이 없으면 아직 출시하면 안 되는지`를 더 보수적으로 정리한다.

## PM 한 줄 결론

- 현재 버전은 `파일럿 가능` 수준이다.
- 실제 유료 배포 버전은 `증빙 신뢰성`, `팀 사용성`, `결제 명분`, `운영 안전성`이 더 필요하다.
- 따라서 다음 목표는 `기능 몇 개 추가`가 아니라 `업무 시스템으로 믿고 쓸 수 있는 수준`으로 전환하는 것이다.

## MVP와 Production v1의 차이

### 현재 MVP가 잘하는 것

- 현장 사진과 사유를 빠르게 남긴다.
- 작업 건과 연결한다.
- 변경 견적과 설명 초안을 만든다.
- 합의 기록과 타임라인을 남긴다.

### 아직 부족한 것

- 사장님 혼자만 쓰는 구조다.
- 고객에게 보여주는 합의 링크나 확인 경험이 없다.
- 작업 건 검색, 정렬, 누적 데이터 관리가 약하다.
- 접근 제어가 `공유 코드 1개` 수준이다.
- 장애나 분쟁 시 운영자가 신뢰할 수 있는 감사 로그와 백오피스가 부족하다.

## Production v1 제품 목표

- `입주청소 사장님 1인 + 실장/직원 1~3명`이 함께 쓸 수 있어야 한다.
- `현장 추가금 설명`이 말로만 끝나지 않고 `고객 확인 흔적`까지 남아야 한다.
- 단순 기록이 아니라 `계약률 방어`, `분쟁 감소`, `재설명 시간 단축`이 수치로 보여야 한다.
- 유료 전환 이유가 명확해야 한다.

## v1 핵심 사용자 가치

1. 현장에서 30초 안에 추가 작업 근거를 남길 수 있다.
2. 고객에게 보내는 설명이 매번 흔들리지 않는다.
3. 누가, 언제, 어떤 금액과 사유로 설명했는지 다시 찾을 수 있다.
4. 합의/보류/제외 데이터가 쌓여 다음 견적 정확도가 올라간다.

## v1 필수 기능

### 1. 조직/권한

- 사업장 단위 계정
- OWNER / MANAGER / STAFF 역할
- 작업 건 생성/수정/열람 권한 분리
- 직원별 활동 로그

### 2. 고객 확인 경험

- 고객용 설명 링크 생성
- 고객이 모바일에서 사진, 추가 작업, 변경 금액을 한 화면에서 확인
- 고객 확인 버튼 또는 확인 회신 기록 저장
- 링크 만료 시간과 재발송 기능

### 3. 현장 업무성 강화

- 현장 기록 템플릿 즐겨찾기
- 자주 쓰는 사유/문구 프리셋
- 작업 건 중복 방지
- 주소/고객명 검색 개선
- 최근 작업 건 빠른 재진입

### 4. 운영/증빙 강화

- 수정 이력과 감사 로그
- 삭제 대신 soft delete
- 합의 기록 변경 이력 보존
- 백업/복구 UI
- 관리자 운영 페이지

### 5. 리포트와 결제 명분

- 월간 합의 성공률
- 보류/제외 사유 분포
- 추가금 방어 총액
- 초안 사용률
- 팀별 처리 속도

## v1에서 일부러 제외할 것

- 범용 CRM 전체 대체
- 회계/세금계산서 기능
- 복잡한 전자계약 기능
- 여러 업종 동시 지원
- AI 자동 음성 요약까지 한 번에 확장

## 추천 첫 유료 패키지

### Basic

- 1개 사업장
- 사용자 1명
- 월 작업 건 제한
- 기본 증빙과 설명 초안

### Team

- 사용자 3~5명
- 권한 분리
- 고객 확인 링크
- 백업/운영 로그
- 리포트 제공

## PM 기준 v1 성공 지표

- 주간 활성 사업장 수
- 작업 건 대비 합의 기록 저장률
- 설명 초안 생성 후 실제 복사/전달 비율
- 고객 확인 링크 열람률
- 보류 상태에서 재설명 후 합의 전환률
- 유료 전환율
- 4주 유지율

## 우선순위 로드맵

### Phase 1. Pilot Plus

- 한글/입력 품질 안정화
- 작업 건 검색 강화
- 리셋 없는 운영 흐름
- 운영자 점검 화면

### Phase 2. Production Beta

- 사업장 계정
- 사용자 로그인
- 고객 확인 링크
- 감사 로그
- 기본 리포트

### Phase 3. Paid Launch

- 결제
- 플랜 제한
- 팀 권한
- 고객지원 운영 체계
- 장애 대응과 SLA 초안

## 출시 판단

- 아래가 모두 충족될 때만 `실제 유료 배포`로 본다.
- 파일럿에서 3개 이상 사업장이 반복 사용한다.
- 작업 건 대비 합의 기록 저장률이 안정적으로 나온다.
- 사용자들이 `카톡 설명 시간 줄었다`를 반복적으로 말한다.
- 접근 코드가 아니라 정식 로그인/권한 구조로 전환됐다.
- DB/백업/로그 체계가 운영 가능한 수준이다.
