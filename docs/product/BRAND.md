# BRAND

## Brand Decision

- Service name: `다밋`
- English wordmark: `DAMIT`
- Recommended product slug: `damit`
- Recommended staging project slug: `damit-staging`
- Recommended production project slug: `damit-prod`

## PM Judgment

- `GO` for a stronger, product-level visual identity

The product has moved beyond internal prototyping. The current UI is functional, but too close to a generic AI SaaS aesthetic. We should now express a clearer product identity before adding more feature surface.

## Brand Position

`다밋` is not a chatbot brand.  
It is an operating surface for messy field conversations, changed estimates, customer explanations, confirmations, and evidence.

The brand should feel like:

- a calm field desk
- a trustworthy operating ledger
- an evidence-first workflow
- a product used by real owners, not a demo tool

## Brand Promise

- Korean: `현장 설명은 빠르게, 운영 기록은 분명하게`
- Short line: `설명은 빠르게, 기록은 분명하게`
- English helper: `A calmer way to explain field changes and keep the record clear.`

## Tone

- practical over flashy
- grounded over futuristic
- evidence-first over “smart” sounding
- confident over playful
- warm enough for owners, strict enough for operations

## Visual Direction

The visual system should feel closer to:

- paper
- ink
- docket labels
- status stamps
- operating desk surfaces

And farther from:

- glassmorphism
- glowing gradients
- floating AI bubbles
- generic bright-blue SaaS dashboards

## Core Design Principles

1. One calm surface language
- Use fewer visual tricks.
- Prefer strong hierarchy, solid cards, and visible structure.

2. Brand through operation, not decoration
- The product should look like an owner-facing operating tool.
- Decorative elements should support clarity, not compete with it.

3. Evidence-first hierarchy
- Risk, action, and proof should visually outrank secondary metadata.

4. Warm paper, sharp ink
- Use warm background surfaces, deep ink text, and one strong blue for action confidence.
- Use amber as a human signal, not as a large decorative fill.

5. Consistent route identity
- Landing, login, home, workspace, account, ops, and admin should feel like the same system.
- Each route can vary in density, but not in design language.

## Color System

- Ink Navy: trust, structure, primary reading
- Work Blue: action, navigation, confident progress
- Field Amber: caution, human warmth, highlighted prompts
- Paper Sand: background and operating surface warmth
- Moss Green: completion, stability, verified states

## Typography

- Primary Korean UI type: clean, modern, high-legibility sans
- Use uppercase utility labels sparingly as operational markers
- Headings should feel editorial and decisive, not startup-generic

## Interaction Style

- Dense but calm
- Fast to scan
- Strong first action
- Clear terminal states
- Minimal motion, meaningful emphasis
