# PRD

## 제품명

- 브랜드명: `다밋`
- 기능 설명명: `현장 추가금 합의 비서`
- 한 줄 정의: 현장에서 작업 범위가 달라졌을 때 사진, 추가 사유, 짧은 메모를 바탕으로 `변경 견적 설명`과 `고객 합의 내용`을 바로 정리하는 분쟁 방지 도구
- 브랜드 한 줄: `설명은 빠르게, 기록은 분명하게`

## 포지셔닝

### 우리가 파는 것

- 우리는 `추가금 말싸움을 줄이는 현장 합의 레이어`를 판다.
- 우리는 `법률 계약`을 파는 것이 아니다.
- 우리는 `현장 상황 변경을 설명 가능하고 기록 가능한 상태로 정리하는 도구`를 판다.

### 우리가 대체하지 않는 것

- 당근의 `견적 요청`, `채팅`, `비즈프로필`
- 네이버의 `예약`, `톡톡`, `플레이스 관리`
- 카카오의 `채널 상담`, `비즈니스 메시지`
- 범용 CRM, 전자계약, 결제 서비스

### 우리가 해결하는 문제

- 작업 전에는 보이지 않던 오염, 폐기물, 곰팡이, 짐, 구조 차이로 현장 범위가 바뀐다.
- 사장님은 추가금이 필요한 이유를 그 자리에서 설명해야 하지만 증빙과 문구가 부족하다.
- 설명이 감정적으로 흐르면 `매출 손실`, `리뷰 악화`, `분쟁`, `작업 지연`이 생긴다.
- 사진만 있어도 `왜 추가인지`, `원래 포함 범위와 뭐가 다른지`, `고객이 무엇에 동의했는지`가 남지 않으면 분쟁 방어력이 약하다.

## 배경

- 홈서비스 업종에서는 온라인에서 받은 사전 정보와 실제 현장 상태가 자주 다르다.
- 특히 입주청소는 기본 청소 범위와 추가 작업 경계가 자주 오해된다.
- 많은 사장님은 추가금 판단은 할 수 있지만 이를 고객에게 `납득 가능하게 표현`하고 `기록으로 남기는 일`을 어려워한다.
- 메신저에 즉석으로 보낸 설명은 문맥이 약하고, 사진은 남아도 `왜 추가되는지`와 `무엇에 동의했는지`가 구조화되지 않는다.
- 거대 플랫폼은 문의 접점은 제공하지만 `현장 변경 합의` 자체를 돕지는 않는다.

## 목표

### 비즈니스 목표

- 30일 안에 파일럿 고객 2팀 확보
- 45일 안에 실제 현장 사례 20건 이상 수집
- 60일 안에 유료 고객 1곳 이상 또는 유료 의사 3곳 이상 확보

### 제품 목표

- 현장 범위 변경 사유를 1분 안에 정리 가능하게 만든다.
- 사진 기반의 `빠른 현장 기록`을 20초 안에 시작 가능하게 만든다.
- 고객에게 보낼 변경 견적 설명 초안을 30초 내 생성한다.
- 현장 합의 과정을 `설명 가능`하고 `되돌아볼 수 있는 기록`으로 남긴다.

### 사용자 목표

- 추가금 설명 스트레스를 줄인다.
- 감정 소모 없이 고객과 합의를 끝낸다.
- 나중에 분쟁이 생겨도 근거를 다시 꺼낼 수 있다.

## 대상 사용자

### 핵심 타깃

- 업종: `입주청소`
- 규모: 1인 사장 또는 2~5인 소규모 팀
- 특징:
  - 현장 오염도, 짐, 폐기물, 옵션 범위에 따라 견적 변경이 잦다.
  - 고객과 직접 대화하며 바로 설명하고 설득해야 한다.
  - `원래 포함 아닌가요?` 같은 질문을 자주 받고, 추가금 설명을 체계적으로 남기지 못해 갈등을 겪는다.

### 초기 확장 타깃

- 에어컨 청소
- 방충망/블라인드 설치
- 소형 수리/시공
- 부분 철거/폐기물 처리

## 핵심 문제 정의

### 문제 1. 현장 변경은 자주 생기는데 첫 기록을 놓친다

- 현장 문제는 발견 즉시 사진과 사유를 남겨야 하지만, 기본 정보 입력이 먼저 오면 기록을 놓치기 쉽다.

### 문제 2. 사진은 있어도 설명 문맥이 없다

- 사진만 보내면 고객은 `그래서 왜 추가금이 필요한지`와 `원래 포함 범위와 뭐가 다른지`를 이해하지 못한다.

### 문제 3. 합의 상태는 있어도 합의 내용이 남지 않는다

- 설명했는지, 고객이 어떤 채널로 어떤 금액에 동의했는지, 제외를 선택했는지가 구조적으로 남지 않는다.

### 문제 4. 매출 손실과 리뷰 리스크가 동시에 생긴다

- 사장님은 갈등을 피하려고 그냥 손해를 보고 넘어가거나, 반대로 강하게 설명하다가 불만을 만든다.

## 가치 제안

- `문제가 보이면 먼저 사진부터 남기게 해준다`
- `사진 + 사유 + 메모를 고객이 이해할 설명 문장으로 바꿔준다`
- `기본 범위와 추가 작업 차이`를 설명 가능하게 만든다.
- `말싸움 대신 합의 내용과 기록`을 남겨준다.

## Jobs To Be Done

- 현장에서 예상과 다른 상태를 발견했을 때, 먼저 증빙을 남기고 싶다.
- 고객에게 추가 작업 필요성과 금액 변동 이유를 빠르게 설명하고 싶다.
- 고객이 무엇에 동의했는지 나중에도 확인할 수 있게 남기고 싶다.
- 추가금을 포기하지 않으면서도 불필요한 감정 소모는 줄이고 싶다.

## 핵심 사용 시나리오

### 시나리오 1. 문제 발견 직후 빠른 현장 기록

1. 사장님이 현장에서 예상과 다른 오염도나 추가 작업 요소를 발견한다.
2. 고객 정보보다 먼저 사진과 상세 사유를 남긴다.
3. 시스템은 현장 기록을 임시 저장하고 작업 건에 연결한다.

### 시나리오 2. 범위 대비와 고객 설명 초안 생성

1. 시스템은 기본 포함 범위와 추가 작업 항목을 대비해서 보여준다.
2. 사장님은 변경 후 금액을 입력한다.
3. 시스템은 현장 상태, 추가 작업, 범위 차이, 금액 변경을 포함한 고객 설명 초안을 만든다.

### 시나리오 3. 합의 내용 기록

1. 고객과 설명이 끝난 뒤 현재 상태를 `설명완료`, `합의완료`, `보류`, `작업 제외` 중 하나로 기록한다.
2. 확인 채널, 확인 시각, 확정 금액, 고객 반응 메모를 남긴다.
3. 추후 분쟁 시 사진, 사유, 범위 대비, 초안 문구, 합의 이력을 한 화면에서 확인한다.

## 제품 원칙

- `법적 효력`보다 `현장 설득력`을 우선한다.
- 완전 자동화보다 `빠른 초안 + 수동 보정`을 우선한다.
- 범용 업종 확장보다 `입주청소 한 업종 완성도`를 우선한다.
- 예쁜 관리 화면보다 `현장 첫 기록 속도`를 우선한다.
- AI가 똑똑해 보이는 것보다 `사장님이 손해를 덜 보는 것`을 우선한다.

## 핵심 기능 요구사항

### 1. 빠른 현장 기록

- 사진 1장 이상을 가장 먼저 남길 수 있어야 한다.
- 상세 고객 정보가 없어도 임시 저장이 가능해야 한다.
- 입력값:
  - 현장 사진
  - 1차 사유 카테고리
  - 2차 상세 사유 또는 짧은 메모

### 2. 작업 건 연결/보완

- 현장 기록을 기존 작업 건에 연결하거나 새 작업 건으로 만들 수 있어야 한다.
- 기본 입력값:
  - 고객명
  - 연락처 또는 메모용 식별 정보
  - 주소 또는 현장명
  - 원래 견적 금액

### 3. 범위 대비 정리

- 기본 포함 범위와 현장 추가 작업 차이를 보여준다.
- 고객이 `원래 포함 아닌가요?`라고 물을 때 설명 가능한 구조여야 한다.

### 4. 변경 견적 정리

- 변경 전 금액과 변경 후 금액을 입력한다.
- 차액을 자동 계산한다.
- 추가 작업 내용을 고객이 이해할 수 있게 요약한다.

### 5. 고객 설명 초안 생성

- 입력값 기반으로 다음 요소를 포함한 초안을 제공한다.
- 포함 요소:
  - 현장 상태 설명
  - 추가 작업 항목
  - 왜 기본 범위 밖인지 또는 왜 사전 가정과 다른지
  - 금액 변경 내역
  - 작업 진행 여부 확인 요청
- 초안은 복사 가능한 텍스트로 제공한다.

### 6. 합의 기록

- 상태값:
  - 미설명
  - 설명완료
  - 합의완료
  - 보류
  - 작업 제외
- 함께 남길 필드:
  - 확인 채널
  - 확인 시각
  - 확정 금액
  - 고객 반응 메모

### 7. 건별 타임라인

- 사진, 사유, 금액 변경, 생성된 초안, 합의 기록, 상태 변경 기록을 시간순으로 보여준다.

## 비기능 요구사항

- 모바일에서 한 손 조작이 가능해야 한다.
- 첫 현장 기록은 20초 내 시작 가능해야 한다.
- 전체 입력은 1분 이내 완료를 목표로 한다.
- 초안 생성 결과는 10초 이내 확인 가능해야 한다.
- 사진과 입력 내용은 건별로 쉽게 다시 찾을 수 있어야 한다.
- 첫 버전은 고급 AI 판단보다 `일관된 설명 구조`를 우선한다.

## 성공 지표

### 1차 지표

- 주간 빠른 현장 기록 생성 수
- 생성된 건 중 고객 설명 초안 사용률
- 생성된 건 중 합의 기록 완료율
- 작업 제외/거절 사유 기록률
- 파일럿 사용자 유지율
- 유료 고객 수 또는 유료 의사 수

### 2차 지표

- 사용자가 체감한 분쟁 감소 반응
- 사용자가 체감한 추가금 방어 성공 사례 수
- `원래 포함 아닌가요?` 대응이 쉬워졌다는 정성 피드백

## 수익화 가설

- 초기 세팅비 + 월 구독이 가장 현실적이다.
- 가설 요금제:
  - 초기 세팅비 15만~25만원
  - 월 구독 9만~15만원
- 지불 논리:
  - 한 번의 추가금 방어 또는 한 번의 분쟁 감소만으로도 비용 설명이 가능하다.

## GTM 가설

- 첫 고객군은 `입주청소 사장님 커뮤니티`, `당근/카페 활동 사장님`, `지인 연결`이 현실적이다.
- 세일즈 메시지는 `AI`보다 `추가금 설명 스트레스 감소`, `손해 보는 작업 감소`, `합의 기록 확보`에 맞춘다.
- 데모는 반드시 `빠른 현장 기록 -> 범위 대비 -> 설명 초안 -> 합의 기록` 흐름으로 보여준다.

## 주요 가설

- 가설 1: 사장님은 추가금 판단 자체보다 `어떻게 설명할지`에서 더 큰 스트레스를 느낀다.
- 가설 2: 입주청소에서는 `기본 범위 vs 추가 작업` 대비가 있어야 고객 설득력이 올라간다.
- 가설 3: 완전한 전자계약이 없어도 `설명 초안 + 증빙 기록 + 합의 내용`만으로도 충분한 가치를 느낀다.
- 가설 4: 문제 발견 직후 사진부터 남길 수 있어야 실제 사용으로 이어진다.

## 리스크

- 법률 계약 서비스로 오해받을 수 있다.
- 업종마다 추가금 발생 사유가 달라 범용화가 어려울 수 있다.
- 실제로는 고객과 통화 설명이 더 많아 텍스트 초안만으로는 부족할 수 있다.
- 현장 사용자는 입력조차 귀찮아할 수 있다.
- 사진 촬영과 기록이 기존 습관에 안착하지 않을 수 있다.

## 리스크 대응

- 모든 문구에서 `합의 초안`, `증빙 기록`, `설명 도구`라고 명확히 표현한다.
- 첫 버전은 `입주청소` 한 업종만 지원한다.
- 텍스트 초안은 메신저 전송용과 구두 설명용 두 톤으로 나눈다.
- 초기 파일럿은 반수동 운영으로 실제 현장 패턴을 먼저 수집한다.
- 입력 단계를 `사진 -> 사유 -> 금액 -> 합의 기록` 중심으로 최소화한다.

## 출시 판단 기준

### 진행 기준

- 파일럿 3팀 이상이 실제 현장에서 2회 이상 반복 사용한다.
- 현장 사례 20건 이상에서 상세 사유 카테고리가 일정 패턴으로 묶인다.
- 설명 초안 사용 후 `설명하기 편했다` 반응이 5건 이상 나온다.
- 합의 기록 필드가 70% 이상 채워진다.
- 60일 안에 유료 의사 3곳 또는 실제 유료 전환 1곳 이상이 나온다.

### 중단 또는 재정의 기준

- 대부분의 사용자가 그냥 통화로 해결하고 별도 기록 도구를 쓰지 않는다.
- 고객 설명 초안이 실전에서 거의 쓰이지 않는다.
- 입주청소 업종 내에서도 상세 사유 구조가 너무 다양해 템플릿화가 어렵다.
- 제품이 `전자계약` 기대를 강하게 불러와 리스크가 커진다.
