# MVP SCOPE

## MVP 목표

- `현장 추가금 합의 비서`의 핵심 가설을 가장 작은 범위로 검증한다.
- 검증할 것은 `AI가 멋진가`가 아니라 `추가금 설명과 합의 내용 기록이 실제로 현장에서 도움이 되는가`이다.
- 첫 타깃 업종은 `입주청소`로 고정한다.

## MVP 한 줄 정의

- 현장에서 예상과 다른 상태가 발견됐을 때 사진부터 먼저 남기고, 사유와 금액 변경을 정리한 뒤 고객에게 보여줄 `추가금 설명 초안`과 `합의 내용 기록`을 바로 만드는 모바일 우선 도구

## 이번 MVP에서 꼭 증명할 것

### 1. 사용성 가설

- 사장님은 현장에서 `사진 + 상세 사유 선택 + 짧은 메모` 정도는 실제로 입력할 수 있다.

### 2. 가치 가설

- `기본 포함 범위 vs 추가 작업` 대비와 고객 설명 초안이 있으면 설명 스트레스가 줄어든다.

### 3. 지불 가설

- 한 번의 추가금 방어 또는 한 번의 분쟁 감소 사례가 나오면 유료 전환 가능성이 생긴다.

## 대상 사용자

- 업종: 입주청소
- 규모: 1인~3인 팀
- 상황: 현장 상태에 따라 추가 작업과 금액 변경이 자주 생기는 팀
- 현재 방식: 사진을 카톡으로 보내거나 말로 설명하고, 고객이 무엇에 동의했는지는 거의 남기지 않음

## 핵심 시나리오

### 시나리오 A. 빠른 현장 기록

1. 사장님이 현장에서 예상보다 심한 오염, 폐기물, 옵션 외 작업을 발견한다.
2. 앱에서 `빠른 현장 기록`을 열고 사진을 찍는다.
3. 추가 사유를 고르고 필요하면 짧은 메모를 남긴다.
4. 나중에 작업 건과 연결하거나 기본 정보를 보완한다.

### 시나리오 B. 변경 견적 설명 준비

1. 시스템이 기본 포함 범위와 추가 작업을 대비해서 보여준다.
2. 원래 견적 금액과 변경 후 금액을 입력한다.
3. 시스템이 고객 설명용 문구를 만든다.
4. 사장님은 문구를 복사해 메신저로 보내거나 직접 설명에 활용한다.

### 시나리오 C. 합의 내용 기록

1. 고객과 통화 또는 메시지 설명 후 상태를 업데이트한다.
2. 확인 채널, 확인 시각, 확정 금액, 고객 반응 메모를 남긴다.
3. 나중에 같은 건을 열어 사진, 사유, 범위 대비, 금액, 합의 내용을 다시 확인한다.

## In Scope

### 기능

- 빠른 현장 기록 시작
- 작업 건 생성 또는 기존 작업 건 연결
- 기본 현장 정보 입력
- 현장 사진 업로드
- 1차/2차 추가 사유 선택
- 짧은 텍스트 메모 입력
- 기본 포함 범위 요약 표시
- 변경 전/후 금액 입력 및 차액 계산
- 고객 설명 초안 생성
- 합의 내용 기록
- 작업 건 목록/상세 화면
- 건별 타임라인

### 입주청소 전용 입력 필드

- 고객명 또는 현장 식별명
- 연락처 메모
- 주소 또는 현장명
- 원래 견적 금액
- 변경 후 금액
- 추가 사유 1차 카테고리
- 추가 사유 2차 상세 항목
- 현장 메모
- 사진
- 합의 상태
- 확인 채널
- 확인 시각
- 확정 금액
- 고객 반응 메모

### 입주청소 전용 사유 카테고리 예시

- 1차 카테고리:
  - 오염도 문제
  - 제거 작업 추가
  - 공간 범위 추가
  - 구조/평수 차이
  - 폐기물/짐 문제
- 2차 상세 예시:
  - 니코틴 오염
  - 곰팡이 제거
  - 스티커/보양지 제거
  - 폐기물 정리
  - 베란다 추가
  - 다용도실 추가
  - 창고 추가
  - 구조 차이

### 운영 방식

- 초반 설명 초안은 템플릿 중심으로 생성한다.
- 기본 포함 범위도 입주청소 표준 템플릿 형태로 먼저 제공한다.
- 필요 시 운영자가 반수동으로 카테고리와 문구를 보정할 수 있다.
- 고객이 직접 서명하거나 클릭 승인하는 기능은 넣지 않는다.

## Out of Scope

- 전자서명 또는 법적 계약 기능
- 자동 결제 링크 발송
- 당근/카카오/네이버 직접 연동
- 음성 인식 기반 자동 요약
- 멀티 업종 지원
- 팀 권한 체계
- 고급 통계 대시보드
- 자동 리마인드 메시지 발송
- 리뷰 관리 기능
- CRM 전체 파이프라인 관리

## MVP 화면 범위

### 1. 작업 건 목록

- 진행 중인 현장 건과 합의 상태를 빠르게 본다.
- `빠른 현장 기록` CTA가 가장 먼저 보인다.

### 2. 빠른 현장 기록

- 사진 업로드, 1차/2차 사유 선택, 메모 입력을 한다.

### 3. 작업 건 생성/연결

- 고객명, 현장명, 원래 견적을 입력하거나 기존 작업 건에 연결한다.

### 4. 범위 대비/변경 견적/설명 초안 화면

- 기본 포함 범위, 추가 작업, 변경 전/후 금액, 차액, 고객 설명 초안을 보여준다.

### 5. 합의 기록 화면 또는 섹션

- 상태, 확인 채널, 확인 시각, 확정 금액, 고객 반응 메모를 입력한다.

### 6. 작업 건 상세 타임라인

- 사진, 입력 사유, 초안, 합의 기록, 상태 변경 이력을 한 화면에서 확인한다.

## UX 원칙

- 첫 행동은 `사진 확보`여야 한다.
- 선택 중심 UI로 입력 피로를 줄여야 한다.
- 고객이 자주 묻는 `원래 포함 아닌가요?`에 답할 수 있어야 한다.
- 설명 초안은 길고 복잡한 문서가 아니라 바로 보낼 수 있는 메시지 느낌이어야 한다.
- 상태값만 남기지 말고 합의 내용이 함께 남아야 한다.

## 운영 가정

- 파일럿 초기에는 하루 1~5건 수준의 현장 사례로도 충분하다.
- 정확도보다 `설명하기 쉬워졌는지`와 `합의 내용이 남는지`를 더 먼저 본다.
- 설명 초안 품질이 떨어지면 수동 보정 운영으로 학습한다.

## 성공 기준

### 정량

- 파일럿 사용자 3팀 확보
- 실제 현장 사례 20건 이상 기록
- 기록된 사례 중 사진 선기록 비율 70% 이상
- 기록된 사례 중 설명 초안 사용률 60% 이상
- 기록된 사례 중 합의 내용 필드 완성률 70% 이상
- 45~60일 내 유료 의사 3곳 또는 유료 전환 1곳 이상

### 정성

- `추가금 설명하기 편했다` 반응 5건 이상
- `원래 포함 아닌가요?에 답하기 쉬웠다` 반응 3건 이상
- `그냥 말로만 할 때보다 덜 싸웠다` 또는 유사 피드백 3건 이상
- `고객이 무엇에 동의했는지 남겨서 안심됐다` 반응 확인

## 실패 기준

- 사장님이 현장에서 사진 선기록조차 거의 하지 않는다.
- 고객 설명 초안이 실전에서 거의 사용되지 않는다.
- 합의 내용 기록이 반복 사용으로 이어지지 않는다.
- 입주청소 업종에서도 상세 사유 템플릿이 너무 자주 깨진다.
- 제품이 법률 계약 서비스로 오해받아 기대 관리가 어려워진다.

## 릴리즈 전략

### 1단계. 반수동 파일럿

- 입주청소 사장님 2~3팀과 현장 사례를 직접 수집한다.
- 실제 사례를 기반으로 1차/2차 사유 카테고리와 설명 문구 템플릿을 다듬는다.
- 기본 포함 범위 템플릿을 함께 조정한다.

### 2단계. 셀프 입력형 MVP

- 모바일 웹 기반으로 빠른 현장 기록, 작업 건 연결, 금액 입력, 초안 확인, 합의 기록까지 제공한다.
- 운영자는 백오피스 또는 수동 보정 방식으로 품질을 관리한다.

### 3단계. 유료 검증

- 파일럿 고객에게 초기 세팅비 + 월 구독 형태를 제안한다.
- 세일즈 포인트는 `추가금 방어`, `말싸움 감소`, `현장 기록 확보`, `합의 내용 기록` 네 가지로 고정한다.

## 운영 메모

- 첫 데모는 반드시 `빠른 현장 기록 -> 범위 대비 -> 금액 변경 -> 고객 설명 초안 -> 합의 기록` 흐름으로 보여준다.
- 세일즈 문구에서 `전자계약`, `법적 효력`, `자동 분쟁 해결` 같은 표현은 피한다.
- 제품 설명은 `합의 초안`과 `증빙 기록`, `합의 내용 기록`에 집중한다.

## 이후 확장 후보

- 고객 확인 링크 또는 간단한 읽음 기록
- 음성 메모 입력 추가
- 업종 확장: 에어컨 청소, 소형 시공, 설치업
- A/S 접수 트리아지 비서 연동
- 시공 리포트 비서 연동
