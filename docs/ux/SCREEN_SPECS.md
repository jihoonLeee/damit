# SCREEN SPECS

## 문서 목적

- `현장 추가금 합의 비서` MVP의 화면별 목적, 구성 요소, 액션, 상태를 정의한다.
- 첫 버전은 입주청소 업종의 `추가금 설명`, `범위 대비`, `합의 증빙` 흐름만 다룬다.

## Screen 1. 작업 건 목록

### 목적

- 현재 처리 중인 현장 건을 빠르게 확인한다.
- 설명이 필요한 건, 합의가 끝난 건, 제외된 건을 상태 기준으로 찾는다.
- 현장에서 가장 먼저 눌러야 할 `빠른 현장 기록`으로 진입시킨다.

### 주요 사용자 질문

- 지금 바로 설명해야 할 건이 뭐지?
- 거절되거나 작업 제외된 건은 뭐였지?
- 현장에서 사진부터 남기려면 어디로 가야 하지?

### 주요 UI 요소

- 상단 제목: `작업 건`
- 대표 CTA: `빠른 현장 기록`
- 보조 CTA: `새 작업 건`
- 상태 필터 칩:
  - 전체
  - 미설명
  - 설명완료
  - 합의완료
  - 보류
  - 작업 제외
- 작업 건 카드 리스트

### 작업 건 카드 구성

- 고객명 또는 현장명
- 주소 축약
- 원래 견적 / 변경 후 금액
- 대표 상세 사유 1개
- 현재 상태 배지
- 합의 기록 존재 여부
- 최근 업데이트 시각

### 주요 액션

- 빠른 현장 기록 시작
- 새 작업 건 생성
- 필터 변경
- 카드 탭으로 상세 진입

### 빈 상태

- 문구: `문제가 보이면 먼저 사진부터 남겨보세요`
- 대표 CTA: `빠른 현장 기록`
- 보조 CTA: `작업 건 만들기`

### 에러 상태

- 목록 로드 실패 시 재시도 버튼 노출

## Screen 2. 빠른 현장 기록

### 목적

- 현장 문제를 발견한 직후 가장 먼저 증빙을 남긴다.
- 고객 기본정보보다 사진과 사유를 우선 기록한다.

### 주요 UI 요소

- 사진 업로드 영역 또는 카메라 진입
- 1차 사유 카테고리 선택
- 2차 상세 사유 선택
- 짧은 메모 입력창
- 저장 CTA: `기록 저장`

### 입력 규칙

- 사진: 1장 이상 필수
- 1차 사유: 필수
- 2차 상세 사유 또는 메모: 최소 하나 필수
- 메모: 상세 사유를 선택했으면 선택

### 1차 사유 예시

- 오염도 문제
- 제거 작업 추가
- 공간 범위 추가
- 구조/평수 차이
- 폐기물/짐 문제

### 2차 상세 사유 예시

- 니코틴 오염
- 곰팡이 제거
- 스티커/보양지 제거
- 폐기물 정리
- 베란다 추가
- 다용도실 추가
- 창고 추가
- 구조 차이

### 저장 후 동작

- `기존 작업 건 연결` 또는 `새 작업 건 만들기` 선택 화면으로 이동

### 에러 상태

- 사진 업로드 실패
- 사유 미선택
- 저장 실패

## Screen 3. 작업 건 생성/보완

### 목적

- 빠른 현장 기록에 고객/견적 맥락을 붙인다.

### 입력 필드

- 고객명 또는 현장 식별명
- 연락처 메모
- 주소 또는 현장명
- 원래 견적 금액

### 주요 UI 요소

- 상단 제목: `작업 건 연결`
- 옵션 탭 또는 분기:
  - 새 작업 건 만들기
  - 기존 작업 건에 연결
- 저장 CTA: `연결하고 계속`

### 유효성 규칙

- 새 작업 건일 때 고객명 또는 현장 식별명: 필수
- 새 작업 건일 때 주소 또는 현장명: 필수
- 새 작업 건일 때 원래 견적 금액: 필수, 숫자만 허용
- 기존 작업 건 연결 시 검색 또는 최근 목록에서 선택 필수

### 저장 후 동작

- `작업 건 상세`로 이동
- 방금 저장한 현장 기록 카드가 상세 상단 근처에 노출

## Screen 4. 작업 건 상세

### 목적

- 한 작업 건의 설명, 합의, 증빙을 한 화면에서 본다.

### 정보 우선순위

1. 합의 상태 배지와 금액 요약
2. 고객 설명 초안
3. 합의 기록 카드
4. 기본 포함 범위 vs 추가 작업 카드
5. 현장 기록 카드
6. 타임라인

### 주요 UI 요소

- 상단 헤더:
  - 고객명 또는 현장명
  - 주소
  - 상태 배지
- 금액 요약 카드:
  - 원래 견적
  - 변경 후 금액
  - 차액
- 섹션:
  - 고객 설명 초안
  - 합의 기록
  - 범위 대비
  - 현장 기록
  - 타임라인

### 주요 액션

- 변경 금액 입력
- 설명 초안 생성/복사
- 합의 기록 저장
- 상태 변경
- 현장 기록 추가

### 빈 상태

- 현장 기록이 없으면 `사진과 사유를 먼저 남겨주세요`
- 금액이 없으면 `변경 금액을 입력하면 설명 초안을 만들 수 있어요`
- 합의 기록이 없으면 `설명 후 고객 반응을 남겨두세요`

## Screen 5. 범위 대비 / 변경 견적 / 설명 초안

### 목적

- 고객이 가장 많이 묻는 `원래 포함 아닌가요?`에 답할 근거를 보여주고 설명 문구를 만든다.

### 주요 UI 요소

- 기본 포함 범위 카드
- 현장 추가 작업 카드
- 원래 견적 금액 표시
- 변경 후 금액 입력 필드
- 차액 계산 표시
- 설명 초안 생성 버튼
- 생성된 설명 초안 박스
- 복사 버튼

### 기본 포함 범위 카드 예시

- 기본 입주청소 범위 요약
- 사전 가정 기준 문구
- 현장 조건과 다른 지점 요약

### 현장 추가 작업 카드 예시

- 추가 작업 항목
- 선택한 상세 사유
- 왜 추가 시간이 드는지 또는 왜 별도 작업인지

### 설명 초안 포함 요소

- 현재 현장 상태 요약
- 추가 작업 항목
- 왜 기본 범위 밖인지 또는 왜 사전 가정과 다른지
- 금액 변경 내용
- 진행 여부 확인 요청

### 유효성 규칙

- 변경 후 금액: 필수
- 숫자만 허용
- 설명 초안 생성 전 현장 기록 최소 1건 필요

### 상태

- 생성 전
- 생성 중
- 생성 완료
- 생성 실패

### 생성 실패 시 대응

- `초안을 다시 만들어보세요` 재시도 버튼
- 기존 입력값 유지

## Screen 6. 합의 기록 입력

### 목적

- 고객에게 무엇을 설명했고, 고객이 무엇에 답했는지를 남긴다.

### 주요 UI 요소

- 상태 선택 버튼 그룹:
  - 설명완료
  - 합의완료
  - 보류
  - 작업 제외
- 확인 채널 선택:
  - 현장 대면
  - 전화
  - 카카오톡/문자
  - 기타
- 확인 시각 입력 또는 자동 기록
- 확정 금액 입력
- 고객 반응 메모 입력
- 저장 버튼

### 유효성 규칙

- 상태: 필수
- 합의완료 또는 작업 제외일 때 고객 반응 메모 권장
- 합의완료일 때 확정 금액 필수
- 확인 채널: 필수

### 저장 후 동작

- 상세 상단 상태 배지 즉시 업데이트
- 목록 카드에 합의 기록 존재 여부 반영
- 타임라인에 상태 변경 및 합의 기록 이벤트 추가

### UX 원칙

- 상태와 증빙 필드를 분리하지 말고 함께 입력하게 한다.
- 내부 상태만 바꾸고 끝나는 경험이 되면 안 된다.

## Screen 7. 현장 기록 카드

### 목적

- 분쟁 시 가장 먼저 확인할 현장 근거를 압축해서 보여준다.

### 구성 요소

- 대표 사진 썸네일
- 1차 사유
- 2차 상세 사유
- 짧은 메모
- 기록 시각

### 주요 액션

- 추가 사진 보기
- 현장 기록 추가
- 메모 보완

## Screen 8. 타임라인

### 목적

- 나중에 분쟁이 생겼을 때 근거와 결정 흐름을 빠르게 다시 본다.

### 타임라인 항목 종류

- 빠른 현장 기록 저장
- 작업 건 생성/연결
- 변경 견적 입력
- 설명 초안 생성
- 합의 기록 저장
- 상태 변경
- 추가 메모

### 표시 규칙

- 최신순으로 보여준다.
- 각 항목에 시각과 핵심 요약을 노출한다.
- `작업 제외` 또는 거절 사유는 강조 표시할 수 있다.

## 공통 컴포넌트

### 상태 배지

- 미설명: 회색
- 설명완료: 파랑
- 합의완료: 초록
- 보류: 주황
- 작업 제외: 진한 회색

### 금액 카드

- 원래 견적
- 변경 후 금액
- 차액
- 확정 금액

### 경고 문구

- 법적 효력을 암시하지 않는 중립적 표현 사용
- 예: `고객과 나눈 설명과 반응을 기록해두세요`

## 공통 에러/예외 문구 원칙

- 기술 원인보다 다음 행동을 먼저 제시한다.
- 예:
  - `사진을 다시 올려주세요`
  - `상세 사유를 하나 더 골라주세요`
  - `변경 후 금액을 입력해주세요`
  - `확인 채널을 선택해주세요`

## 반응형 우선순위

- 모바일 세로 화면 최우선
- 빠른 현장 기록은 한 손 조작을 우선한다.
- 상세 화면은 긴 스크롤 1개로 유지한다.
- 데스크톱은 카드 폭 확장 정도만 허용한다.

## 화면 설계 결론

- MVP는 화면 수보다 `기록을 어디서 시작하느냐`와 `합의 내용을 어디까지 남기느냐`가 중요하다.
- 따라서 `빠른 현장 기록`, `범위 대비`, `합의 기록` 세 축이 실제 화면의 중심이 되어야 한다.
