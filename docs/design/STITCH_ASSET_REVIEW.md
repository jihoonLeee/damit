# Stitch Asset Review

## 목적
- `output/stitch` 아래에 생성된 데스크톱/모바일 시안을 현재 `다밋` 제품에 무리 없이 흡수할 수 있는지 검토한다.
- 화면을 그대로 덮어쓰지 않고, 제품 구조와 운영 흐름을 유지하면서 디자인 시스템과 핵심 레이아웃 패턴만 가져오는 것을 원칙으로 한다.

## 전체 판단
- 결론: `직접 치환`보다 `디자인 시스템 흡수 + 핵심 표면 재구성`이 적합하다.
- 이유:
  - Stitch 시안은 `Digital Ledger` 방향이 분명하고 브랜드 톤이 현재 `다밋`과 잘 맞는다.
  - 그러나 `사이드바 중심 앱 셸`, `정적 데모용 이미지`, `목업 전용 카드 밀도`는 현재 제품 구조와 바로 맞지 않는다.
  - 특히 `/app`과 `/ops`는 실제 데이터와 상호작용이 많아, 시안 구조를 통째로 이식하면 오히려 사용성이 떨어질 수 있다.

## Stitch에서 가져올 것
- `문서형 운영 데스크` 톤
  - 따뜻한 종이 배경
  - 짙은 잉크색 타이포그래피
  - 강한 외곽선과 각진 카드
- `에디토리얼 위계`
  - 큰 serif headline
  - 라벨/상태용 compact label
  - 문단보다 제목 중심으로 읽히는 구조
- `운영 패널 구성 방식`
  - verdict/focus 패널
  - status strip
  - focus card + supporting card 구조
- `행동 유도 방식`
  - primary CTA 1개를 강하게
  - 보조 CTA는 선형 버튼으로 분리

## Stitch에서 그대로 가져오지 않을 것
- 좌측 고정 사이드바 기반 앱 셸
- 화면 안에 포함된 샘플 사진/아바타
- Tailwind CDN 기반 정적 마크업
- 과도하게 많은 상단 내비게이션 항목
- 현재 제품 구조에 없는 메뉴 이름

## 화면별 적용 판단

### 바로 반영 가치가 높은 화면
- `랜딩`
- `시작하기`
- `로그인`
- `운영 콘솔`

### 구조는 유지하고 스타일/위계만 흡수할 화면
- `운영 홈`
- `작업 워크스페이스`
- `사장님 마이페이지`
- `시스템 관리자`
- `고객 확인 페이지`

## 현재 프로젝트에서 함께 고칠 문제
- `start.html`, `account.html`, `admin.html`에 남아 있는 한글 깨짐
- 일부 `account.js`, `admin.js`의 깨진 사용자 문구
- 현재 `styles.css`의 둥근 카드/그라디언트/섀도우 중심 톤

## 적용 원칙
1. 기능 흐름은 유지한다.
2. 시안의 구조보다 디자인 시스템을 우선 흡수한다.
3. `다밋` 고유 브랜드 톤을 해치지 않는 범위에서만 반영한다.
4. 한 번에 전부 바꾸지 말고, 핵심 표면부터 단계적으로 반영한다.
