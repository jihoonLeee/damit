# STITCH_AI_USAGE_NOTE

## PM guidance

Do not ask Stitch AI for all screens at once without a shared brand brief.

Use this order:

1. global direction
2. landing
3. login
4. home
5. app
6. ops
7. account
8. admin
9. customer confirmation
10. logo exploration

## Why

If Stitch generates every screen independently, the result often looks:

- too template-like
- too AI-made
- too inconsistent from route to route

The right workflow is:

- lock the visual system first
- lock the route family second
- refine the logo after the product family already has a clear tone

## PM review standard

Reject any result that feels:

- like a startup landing template
- like a generic AI dashboard
- like a design system demo
- prettier than it is useful

Keep any result that feels:

- editorial
- operational
- calm
- credible
- specific to `다밋`
