# STITCH_AI_PROMPT_PACK

## PM goal

Create a full prompt pack for Stitch AI so the next visual exploration strengthens `다밋` instead of producing another generic AI-SaaS interface.

This pack is designed for:

- calmer, more ownable brand direction
- higher UX clarity
- lower "AI-generated template" feeling
- stronger consistency across all product surfaces

---

## 1. Global direction prompt

Use this first before any screen-specific prompt.

### Base prompt

Design a Korean B2B operations product called `다밋 (DAMIT)`.

This product is not a chatbot, not a futuristic AI dashboard, and not a startup landing page. It is an owner-facing operating tool for messy field work: changed estimates, customer explanation, confirmation, agreement logging, and evidence tracking.

The brand should feel like:

- a calm field desk
- an evidence ledger
- a clipped operations folder
- stamped status labels
- a trustworthy owner tool

The visual system should feel:

- warm, grounded, paper-based
- structured and editorial
- calm but confident
- premium without looking luxurious
- practical rather than flashy

Use these design principles:

- warm paper backgrounds, not cold white-blue gradients
- deep navy ink typography
- one strong work-blue for action confidence
- amber only as a caution or human follow-up signal
- green only for closure, verified proof, or stable states
- strong card borders, document-like panels, visible structure
- minimal blur, almost no glassmorphism
- no floating neon AI bubbles
- no generic SaaS dashboard look
- no decorative illustrations that compete with the first action
- no purple-heavy palette

Typography should feel editorial and decisive. Use a Korean product style that feels closer to a modern business newspaper or operating file than a startup template.

Create layouts that feel intentionally designed by a strong product team, not AI-generated.

The product runs on desktop and mobile. Mobile should preserve hierarchy, not simply shrink everything.

Output should feel like one visual family across:

- landing
- start
- login
- home
- app workspace
- customer confirmation
- operations console
- owner account page
- internal admin page

### Negative prompt

Avoid:

- glossy AI SaaS hero sections
- glass cards everywhere
- giant rounded blobs
- over-animated dashboards
- excessive icon noise
- futuristic sci-fi UI
- overly playful consumer app styling
- random abstract gradients with no structural meaning
- visual styles that look interchangeable with hundreds of AI startup templates

---

## 2. Shared art direction

Use these rules in every screen prompt.

### Color direction

- Background: warm sand / paper beige
- Main text: deep navy ink
- Action: confident cobalt blue
- Caution: muted amber
- Success: moss green
- Lines: visible but elegant paper-border lines

### Surface direction

- cards should feel like clipped paper panels
- stronger top card per page for the main focus
- compact uppercase mini labels should read like docket markers
- badges should feel like status stamps, not decorative chips
- modules should feel aligned to a shared document grid

### Spacing direction

- strong rhythm between sections
- fewer but more deliberate layers
- avoid "wall of equal cards"
- primary card should always win visually

### Interaction tone

- decisive primary action
- obvious next step
- closed states should feel clearly finished
- warning states should feel precise, not loud

---

## 3. Stitch instructions for output quality

Add these expectations to every Stitch request.

- Write all UI copy in Korean.
- Use realistic product copy, not lorem ipsum.
- Show the exact product structure rather than generic placeholders.
- Make the design look shippable, not conceptual.
- Maintain consistency across desktop and mobile versions.
- Use subtle, ownable details like ruled lines, paper tabs, clipped labels, and evidence blocks.
- Make sure spacing between cards is intentional and consistent.
- Avoid the feeling that every section was generated independently.

## 4.5 Main app workspace prompt

Design the main DAMIT operating workspace.

This is the densest and most important screen in the product. It manages:

- field capture
- case binding
- revised quote
- customer explanation draft
- customer confirmation
- agreement record
- timeline

Goals:
- make the workflow obvious
- reduce confusion about what to do next
- make terminal states feel truly finished
- keep heavy information readable on both desktop and mobile

Required layout:
- header with operating context
- workflow strip showing the sequence
- operator signal board
- "지금 바로 할 일" section
- three-column or structured workspace:
  - field capture / case binding
  - case list
  - selected case detail

Selected case detail must include:
- current stage card
- focus / next-action card
- revised quote card
- explanation draft card
- customer confirmation card
- agreement card
- field records
- timeline rail

Critical UX rules:
- if a case is AGREED, make it feel closed and review-only
- if a case is EXCLUDED, make it feel closed and non-primary
- long timeline should feel bounded and manageable
- warnings should be short and instantly scannable
- if coming from ops handoff, explain why this case was opened now

Visual direction:
- evidence workspace, not dashboard
- paper-card stack with strong hierarchy
- action blue only where commitment matters
- caution amber only where human follow-up matters

Avoid:
- equal visual weight for all cards
- long dense text blocks
- generic kanban/task app visuals

## 6. Logo direction

PM recommendation:

The logo should not try to look like an AI company mark.

The best direction for `다밋` is:

- document mark
- docket tab
- stamped seal
- ink block + status dot
- evidence folder silhouette

Avoid:

- sparkles
- chat bubbles
- neural nodes
- abstract infinite loops
- generic geometric startup marks

### Recommended logo concept

Create a logo for a Korean B2B operations product named `다밋 (DAMIT)`.

The logo should feel like a mark you might see on an evidence folder, field ledger, or operating docket.

Requirements:

- simple enough to scale to app icon size
- premium but restrained
- not playful
- not futuristic
- should pair well with warm paper backgrounds
- should feel credible in both product UI and documents

Visual cues:

- a document corner, tab, clip, docket edge, or stamped circle
- deep navy as the main color
- one small amber accent
- optional hidden `D` structure inside the mark

Avoid:

- literal broom, house, robot, sparkle, brain, or chatbot icons
- overly rounded toy-like shapes
- generic monoline startup marks

### Logo prompt for image generation

Design a refined brand mark for a Korean B2B field-operations product named `다밋 (DAMIT)`.

The mark should feel like a modern evidence folder or operating ledger symbol, not a tech startup icon. Use deep navy as the main color and a small muted amber accent. The shape should feel structured, calm, trustworthy, and slightly editorial. It may contain a subtle hidden `D`, but should not become a literal lettermark. The result should work as an app icon, favicon, and product mark on warm paper-colored UI surfaces.

Avoid chat bubbles, sparkles, AI motifs, neural graphics, shiny gradients, 3D effects, and generic SaaS startup aesthetics.

### Logo prompt for vector refinement

Create a flat vector logo for `다밋 (DAMIT)` with these qualities:

- solid shapes
- minimal colors
- high contrast on warm beige and white backgrounds
- visually balanced at 24px, 48px, and 128px
- one strong structural silhouette
- subtle evidence-folder or docket-tab character

The logo should look like a product mark designed by a serious software brand, not an AI-generated icon pack.

   - landing
   - login
   - home
3. Lock the brand direction
4. Then generate:
   - app workspace
   - ops
   - account
   - admin
   - customer confirmation
5. Review mobile versions after desktop
6. Only after the full family feels consistent, generate refined logo directions

## 8. PM review checklist for Stitch outputs

Use this after Stitch returns concepts.

- Does it still look like a generic AI SaaS?
- Do all surfaces feel like one product family?
- Does the first action win visually?
- Does the app workspace look like an evidence workflow?
- Does the ops screen feel like dispatch, not analytics?
- Does the account page feel like owner control, not settings clutter?
- Does admin feel constrained and internal?
- Does mobile preserve hierarchy?
- Does the logo feel like an operating mark rather than a startup symbol?

If any answer is no, refine before implementation.
